VMware Kubeapps

The Kubeapps project provides a dashboard for Helm in your cluster, giving you a UI for managing and deploying helm charts.

Kubeapps works similarly to other cluster management dashboards in that it takes the user's token and uses it against the managed Kubernetes cluster. It supports OpenID Connect natively via an integrated oauth2 proxy sidecar. This integration should be skipped with OpenUnison because the OAuth2 proxy project does not work well with short lived tokens. The OAuth2 proxy integrates with OpenID Connect Identity Providers, and does support refresh tokens, but it stores all session data in an encrypted cookie that it only checks to refresh periodically. With OpenUnison's short lived tokens (1-2 minutes depending on clock skew), OAuth2 Proxy ends up re-authenticating users on almost every request.

Integrating Kubeapps with OpenUnison is easier to do via OpenUnison's integrated reverse proxy that injects the user's token into each request. This continues to provide the security of short lived tokens while giving Kubeapps users a secure way to access their dashboard. The OpenUnison project has a helm chart that simplifies integration of Kubernetes management applications so you don't need to create all of the individual objects. Use the below as a template for a values file for this chart:

app:
  label: Kubeapps
  badgeUrl: https://kubeapps.apps.192-168-2-90.nip.io/
  injectToken: true
  proxyTo: http://kubeapps.kubeapps.svc${fullURI}

You can add app.az_groups to allow specific groups instead of relying on just RBAC for authorization:

app:
  az_groups:
  - admins

Once you have updated your values.yaml, you can deploy the chart:

$ helm install kubeapps tremolo/kubernetes-management-app -n openunison -f /path/to/values.yaml

Finally, create an Ingress object that points to OpenUnison with your Kubeapps host name, as an example:

---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  annotations:
    kubernetes.io/ingress.class: nginx
    nginx.ingress.kubernetes.io/affinity: cookie
    nginx.ingress.kubernetes.io/backend-protocol: HTTPS
    nginx.ingress.kubernetes.io/secure-backends: "true"
    nginx.ingress.kubernetes.io/session-cookie-hash: sha1
    nginx.ingress.kubernetes.io/session-cookie-name: openunison-orchestra
    nginx.org/ssl-services: openunison-orchestra
    cert-manager.io/cluster-issuer: enterprise-ca
  name: kubeapps
  namespace: openunison

spec:
  rules:
  - host: kubeapps.apps.192-168-2-90.nip.io
    http:
      paths:
      - backend:
          service:
            name: openunison-orchestra
            port:
              number: 443
        path: /
        pathType: Prefix
  tls:
  - hosts:
    - kubeapps.apps.192-168-2-90.nip.io
    secretName: ou-tls-certificatex
