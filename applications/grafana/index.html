Grafana Community Edition

Grafana is a staple for Kubernetes and other cloud native systems dashboards. The community edition supports OpenID Connect for authentication. You can't map to groups or teams though, only to one of the three roles (Admin, Editor, or Viewer). See Grafana's role documentation for details on what these roles can do.

OpenUnison Configuration

The first step is to add a key to the orchestra-secrets-source Secret in the openunison namespace. This key (in our examples called grafana) should be long and not contain any ampersands (&). It's recommended to use a 64 character random string of numbers and letters.

Next, create a JavaScript mapping that will generate a roles attribute for you. The below example maps all members of the k8s-admins group to the roles claim:

---
apiVersion: openunison.tremolo.io/v1
kind: JavaScriptMapping
metadata:
  namespace: openunison
  name: grafana-role-mapping
spec:
  javascript: |-
    function doMapping(user,name) {
      Attribute = Java.type("com.tremolosecurity.saml.Attribute");

      // get the current groups from the user
      var groups = user.getAttribs().get("groups").getValues()

      // grafanaGroups
      grafanaGroups = new Attribute(name);


      // convert groups from LDAP DNs from Active Directory to standard group names
      for (var i=0;i<groups.length;i++) {
        var group = groups.get(i);

        if (group == 'k8s-admins') {
          grafanaGroups.getValues().add('Admin');
        }
      }



      return grafanaGroups;
    }

Adjust this mapping to suit your needs. Next, you'll need to deploy an identity provider for grafana. This configuration is a good starting point:

---
apiVersion: openunison.tremolo.io/v2
kind: Application
metadata:
  labels:
    app.kubernetes.io/component: openunison-applications
    app.kubernetes.io/instance: openunison-orchestra-login-portal
    app.kubernetes.io/name: openunison
    app.kubernetes.io/part-of: openunison
  name: grafana
  namespace: openunison
spec:
  azTimeoutMillis: 3000
  cookieConfig:
    cookiesEnabled: true
    domain: '#[OU_HOST]'
    httpOnly: true
    keyAlias: session-unison
    logoutURI: /logout
    scope: -1
    secure: true
    sessionCookieName: tremolosession
    timeout: 900
  isApp: false
  urls:
  - azRules:
    - constraint: (groups=k8s-admins)
      scope: filter
    filterChain: []
    hosts:
    - '#[OU_HOST]'
    idp:
      className: com.tremolosecurity.idp.providers.OpenIDConnectIdP
      mappings:
        map:
        - sourceType: user
          targetAttributeName: sub
          targetAttributeSource: sub
        - sourceType: composite
          targetAttributeName: name
          targetAttributeSource: ${givename} ${sn}
        - sourceType: user
          targetAttributeName: preferred_username
          targetAttributeSource: uid
        - sourceType: user
          targetAttributeName: email
          targetAttributeSource: mail
        - sourceType: custom
          targetAttributeName: roles
          targetAttributeSource: com.tremolosecurity.mapping.JavaScriptMapping|k8s,openunison,grafana-role-mapping
        strict: true
      params:
        jwtSigningKey: unison-saml2-rp-sig
        k8sNameSpace: 'openunison'
        k8sTarget: k8s
        sessionStoreClassName: com.tremolosecurity.oidc.k8s.K8sSessionStore
      trusts:
      - name: 'grafana'
        params:
          accessTokenSkewMillis: "120000"
          accessTokenTimeToLive: '60000'
          authChainName: login-service
          clientID: grafana
          codeLastMileKeyName: lastmile-oidc
          codeTokenSkewMilis: '60000'
          publicEndpoint: "false"
          redirectURI: 
          - https://grafana.cluster.tremolo.dev/login/generic_oauth
        secretParams:
        - name: clientSecret
          secretName: orchestra-secrets-source
          secretKey: grafana
    results:
      auFail: default-login-failure
      azFail: default-login-failure
    uri: /auth/idp/grafana

Make sure to update spec.idp.trusts[name:grafana].redirectURI with the redirect URL for your Grafana instance. This value MUST be the same one used for root_url in the [server] section of your grafana.ini file.

Also, update spec.azRules to lineup with your authorization rules for accessing the dashboard.

Next, configure Grafana for OpenID Connect integration into OpenUnison.

Grafana Configuration

First, create a Secret with your clientid (grafana) and the client secret you created for OpenUnison. For instance:

apiVersion: v1
kind: Secret
metadata:
  creationTimestamp: "2023-05-17T20:09:52Z"
  name: grafana-oidc
  namespace: default
  resourceVersion: "393530"
  uid: 833826fd-3e95-4b94-973a-526dd08f8056
type: Opaque
data:
  clientid: Z3JhZmFuYQ==
  clientsecret: M2dpeGdkcTJqOHg3aDZ3a2dsdDMxZnNsaHF5cmxjMnAyaWs3dzk3dTE1ZTF5aXpsanEwNTlsYW9uenNlZGpycw==

DO NOT USE THIS EXACT CLIENTSECRET

Once the Secret is created, the next step is to update your Grafana helm chart. Assuming you're using the Grafana Community helm chart, update your values.yaml with the below. Update for your environment:

extraSecretMounts:
- name: oidc-secrets
  secretName: grafana-oidc
  defaultMode: 0400
  mountPath: /etc/secrets/auth_generic_oauth
  readOnly: true

grafana.ini:
  server:
    root_url: https://grafana.cluster.tremolo.dev
  auth.generic_oauth:
    enabled: "true"
    client_id: $__file{/etc/secrets/auth_generic_oauth/clientid}
    client_secret: $__file{/etc/secrets/auth_generic_oauth/clientsecret}
    name: OpenUnison
    auto_login: "true"
    scopes: openid profile
    use_pkce: "true"
    allow_sign_up: "true"
    auth_url: https://k8sou.cluster.tremolo.dev/auth/idp/grafana/auth
    token_url: https://k8sou.cluster.tremolo.dev/auth/idp/grafana/token
    api_url: https://k8sou.cluster.tremolo.dev/auth/idp/grafana/userinfo
    role_attribute_path: roles

Once updated, you can now login to Grafana and get the correct role.
